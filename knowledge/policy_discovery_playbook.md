# 政策发现与候选事件搜集手册

目标：让 Codex 不只是“想方法”，还要系统性地发现可研究政策、核验政策细节，并判断是否具备可信的反事实设计。

## 1. 候选政策来源

优先从下列来源发现事件：

- 官方法律法规库、政府公报、部门通知、地方政府政策文件库；
- 统计局、生态环境、交通、教育、财政、市场监管、医保、人社等部门数据门户；
- 国际组织和监管机构政策数据库；
- 平台治理规则、API 政策变更、开发者公告、商家规则归档；
- 学术论文和工作论文中的政策冲击，但必须回到原始政策文件核验；
- 新闻报道只作为线索，不作为最终证据。

## 2. 搜索策略

Codex 处理新问题时，先构造三组搜索词：

1. 政策名称 + 发布机构 + “通知/办法/条例/公告/公报/实施”；
2. 政策机制 + 地区/行业 + “试点/分批/资格/阈值/补贴/限制”；
3. 结果变量 + 数据口径 + “统计年鉴/开放数据/API/面板/微观数据”。

对英文或跨国政策，加入 `official`, `gazette`, `regulation`, `effective date`, `implementation`, `dataset`, `microdata`, `panel` 等词。

## 3. 候选事件评分

每个候选政策按 0–2 分打分：

| 维度 | 问题 |
|---|---|
| 时间清晰 | 是否有明确发布日期、生效日期、执行日期、修订/废止日期？ |
| 处理组清晰 | 哪些地区、机构、企业或个人受到政策影响？ |
| 对照组可得 | 是否存在未受影响或晚受影响对象？ |
| 数据可得 | 结果变量和协变量是否能按单位-时间获得？ |
| 机制可信 | 政策是否可能影响目标结果，且不会被同时政策完全混淆？ |
| 溢出可控 | 对照组是否可能受到跨区、市场或网络溢出？ |
| 政策重要性 | 是否具有理论或现实政策意义？ |

总分低于 8 的候选事件通常先做资料搜集，不直接进入估计。

## 4. 研究事件卡片

每个候选事件至少记录：

- policy_name；
- jurisdiction；
- issuing_agency；
- publication_date；
- effective_date；
- legal_form；
- affected_units；
- comparison_units；
- likely_outcomes；
- candidate_design；
- primary_sources；
- data_sources；
- unresolved_verification_items。

## 5. 进入因果评估的最低条件

1. 至少一个 primary policy source 被记录；
2. 处理时间和覆盖范围可操作化；
3. 可提出一个明确 estimand；
4. 有可行结果数据路径；
5. 至少一个识别策略的关键假设可以被诊断或部分检验。
