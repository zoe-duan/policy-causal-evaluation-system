# 政策与数据来源清单

这是一个可扩展 inventory，用来指导 Codex 搜集政策和结果数据。具体项目必须补充真实链接、访问日期和文件哈希。

## 政策原文来源类型

| 类型 | 示例 | 用途 | 风险 |
|---|---|---|---|
| 法律法规/政府公报 | 国家或地方官方公报、法律法规数据库 | 核验法律依据、发布日期、生效日期 | 页面迁移、修订版本混淆 |
| 部门通知/办法 | 财政、交通、生态环境、教育、市场监管等部门 | 确认执行细则、补贴标准、监管范围 | 只找到征求意见稿而非正式稿 |
| 试点名单 | 试点城市、试点学校、示范企业名单 | 构造处理组和事件时间 | 名单公布时间不等于实施时间 |
| 预算/补贴文件 | 财政预算、专项资金、拨付公告 | 构造强度变量 | 拨付和实际使用可能不同步 |
| 执法/处罚公告 | 行政处罚、抽检、监管行动 | 构造执行强度 | 披露偏差、选择性执法 |
| 平台规则公告 | 开发者公告、商家规则、审核政策 | 平台治理和数字经济评估 | 历史版本难找、灰度发布 |

## 结果数据来源类型

| 结果类型 | 常见数据 | 备注 |
|---|---|---|
| 环境 | 监测站、卫星、城市月度污染 | 注意天气、产业结构、监测口径变化 |
| 交通 | 拥堵、出行、事故、车辆登记 | 注意跨区出行和替代行为 |
| 劳动 | 就业、工资、岗位发布、社保 | 注意宏观冲击和地区趋势 |
| 教育 | 入学、成绩、辍学、学校资源 | 注意选择进入和考试制度变化 |
| 企业 | 注册、退出、投资、销售、创新 | 注意行业趋势和政策叠加 |
| 消费/平台 | 交易、商家、价格、评价 | 注意平台规则和算法变更 |

## Codex 输出要求

每次 source scouting 至少输出：

1. source name；
2. source type；
3. primary/secondary；
4. URL 或本地路径；
5. expected fields；
6. access constraints；
7. refresh/version issue；
8. how it maps to treatment/outcome/covariates。
